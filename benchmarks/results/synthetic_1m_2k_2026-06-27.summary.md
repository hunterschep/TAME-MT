# Synthetic 1M / 2k Stress Benchmark

This is a release stress benchmark, not a universal performance claim. The raw
corpora are generated by `benchmarks/bench_synthetic.py` and are not committed.

Machine: Apple M3 Max, 36 GB RAM, macOS-26.5.1-arm64-arm-64bit, Python 3.12.5,
Rust 1.96.0. Git HEAD at measurement:
`99ae9140fdeeb04f34f764c3a46164693613b776` with a dirty worktree containing the release-hardening changes.

| Run | Backend | Fresh Audit | Peak RSS | Threads | Index Build | Indexed Audit | Load + Audit | Cached Score | Prepared Cached | Batch Cached | Index Size |
| --- | --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| audit only | `native_exact` | 45.44s | 4036 MB | 8 | - | - | - | - | - | - | - |
| staged | `native_exact` | 45.50s | 4004 MB | 8 | 39.79s | 23.43s | 47.33s | 0.45s | 0.19s | 0.16s/system | 739 MB |

The exact 1M/2k audit passes the 60s stress target on this machine with
`--threads 8`. Peak RSS is under the 4 GiB target for both audit-only and staged
runs after avoiding retained Python normalized training copies in fresh and
persisted-index paths.

Audit-only command:

```bash
python benchmarks/bench_synthetic.py \
  --train-size 1000000 \
  --test-size 2000 \
  --threads 8 \
  --max-seconds 60 \
  --assert-thresholds
```

Staged command:

```bash
python benchmarks/bench_synthetic.py \
  --train-size 1000000 \
  --test-size 2000 \
  --threads 8 \
  --staged \
  --max-seconds 60 \
  --max-index-build-seconds 60 \
  --max-indexed-seconds 60 \
  --max-indexed-audit-seconds 60 \
  --max-cached-seconds 3 \
  --max-prepared-cached-seconds 1 \
  --max-cached-batch-per-system-seconds 1 \
  --assert-thresholds
```

See `synthetic_1m_2k_2026-06-27.summary.json` for signatures, backend metadata,
peak RSS, thread count, and full stage timings.
